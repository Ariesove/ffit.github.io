##👱 GET 和 post 本质区别

叙述思路

  1. 首先属于 HTTP 协议—–》 引出本质是 TCP 连接——》 引出 tcp 连接次数问题—–> 由次数问题：想到幂等。
  2. 其次想到安全幂等，GET 安全幂等，POST 不安全也不幂等。
  3. 其次想到想起它们属于 HTTP 协议，也就是说也是基于 TCP 链接的，所以它们的本质都是 TCP 连接。
  4. GET 请求是一个 tcp 报文 :(对于 GET 方式的请求，浏览器会把 http header 和 data 一并发送出去，服务器响应 200（返回数据）。) //你请求肯定要有请求头吗，对吧
  5. 而 POST 请求是两个 tcp 报文，(浏览器先发送 header，服务器响应 100，浏览器再发送 data，服务器响应 200 ok（返回数据）。)
  6. 所谓安全就是不会改变服务器资源，所谓的**幂**想起次幂，也就是说多次的意思
  7. 所谓的不安全其实就是会改变，服务器的资源，所谓的不幂等就是，多次操作结果不相同。

💯 100 continue

  1. HTTP 100 Continue 信息型状态响应码表示目前为止一切正常, 客户端应该继续请求, 如果已完成请求则忽略.// 自己的话就是:👌 因为是 100 分所以目前状态正常，可以继续请求。
